Prendre une capture d'écran du Canvas

Dans le navigateur, il existe effectivement 2 fonctions qui permettent de prendre une capture d'écran. L'ancienne canvas.toDataURL et la nouvelle, meilleure canvas.toBlob

Donc, vous pourriez penser qu'il serait facile de prendre une capture d'écran en ajoutant simplement du code comme suit :

<canvas id="c"></canvas>
+<button id="screenshot" type="button">Enregistrer...</button>

const elem = document.querySelector('#screenshot');
elem.addEventListener('click', () => {
  canvas.toBlob((blob) => {
    saveBlob(blob, `screencapture-${canvas.width}x${canvas.height}.png`);
  });
});

const saveBlob = (function() {
  const a = document.createElement('a');
  document.body.appendChild(a);
  a.style.display = 'none';
  return function saveData(blob, fileName) {
     const url = window.URL.createObjectURL(blob);
     a.href = url;
     a.download = fileName;
     a.click();
  };
}());

Le problème est que, pour des raisons de performance et de compatibilité, par défaut, le navigateur efface le tampon de dessin d'un canvas WebGL après que vous ayez dessiné dessus.

La solution consiste à appeler votre code de rendu juste avant de capturer.

Dans notre code, nous devons ajuster quelques éléments. Tout d'abord, séparons le code de rendu.

Empêcher l'effacement du canvas

Supposons que vous vouliez permettre à l'utilisateur de peindre avec un objet animé. Vous devez passer preserveDrawingBuffer: true lorsque vous créez le WebGLRenderer. Cela empêche le navigateur d'effacer le canvas. Vous devez également indiquer à three.js de ne pas effacer le canvas.

const canvas = document.querySelector('#c');
-const renderer = new THREE.WebGLRenderer({antialias: true, canvas});
+const renderer = new THREE.WebGLRenderer({
+  canvas,
+  preserveDrawingBuffer: true,
+  alpha: true,
+});
+renderer.autoClearColor = false;

Notez que si vous étiez sérieux au sujet de la création d'un programme de dessin, ce ne serait pas une solution car le navigateur effacera toujours le canvas chaque fois que sa résolution change. Nous changeons sa résolution en fonction de sa taille d'affichage. Sa taille d'affichage change lorsque la taille de la fenêtre change. Cela inclut lorsque l'utilisateur télécharge un fichier, même dans un autre onglet, et que le navigateur ajoute une barre d'état. Cela inclut également lorsque l'utilisateur tourne son téléphone et que le navigateur passe du mode portrait au mode paysage.

Si vous vouliez vraiment créer un programme de dessin, vous devriez rendre dans une texture en utilisant une cible de rendu.

Obtenir les entrées clavier

Tout au long de ces tutoriels, nous avons souvent attaché des écouteurs d'événements au canvas. Alors que de nombreux événements fonctionnent, l'un de ceux qui ne fonctionne pas par défaut est les événements clavier.

Pour obtenir les événements clavier, définissez le tabindex du canvas à 0 ou plus. Par exemple :

<canvas tabindex="0"></canvas>

Cela finit par causer un nouveau problème cependant. Tout élément ayant un tabindex défini sera mis en évidence lorsqu'il aura le focus. Pour corriger cela, définissez son contour CSS de focus à none

canvas:focus {
  outline:none;
}

Pour démontrer, voici 3 canvases : 

<canvas id="c1"></canvas>
<canvas id="c2" tabindex="0"></canvas>
<canvas id="c3" tabindex="1"></canvas>

et du css juste pour le dernier canvas : 

#c3:focus {
    outline: none;
}

Attachons les mêmes écouteurs d'événements à tous :

document.querySelectorAll('canvas').forEach((canvas) => {
  const ctx = canvas.getContext('2d');

  function draw(str) {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText(str, canvas.width / 2, canvas.height / 2);
  }
  draw(canvas.id);

  canvas.addEventListener('focus', () => {
    draw('has focus press a key');
  });

  canvas.addEventListener('blur', () => {
    draw('lost focus');
  });

  canvas.addEventListener('keydown', (e) => {
    draw(`keyCode: ${e.keyCode}`);
  });
});

Remarquez que vous ne pouvez pas obtenir du premier canvas qu'il accepte les entrées clavier. Le deuxième canvas le peut, mais il est mis en évidence. Le 3ème canvas a les deux solutions appliquées.

Rendre le Canvas Transparent

Par défaut, THREE.js rend le canvas opaque. Si vous voulez que le canvas soit transparent, passez alpha:true lorsque vous créez le WebGLRenderer

const canvas = document.querySelector('#c');
-const renderer = new THREE.WebGLRenderer({antialias: true, canvas});
+const renderer = new THREE.WebGLRenderer({
+  canvas,
+  alpha: true,
+});

Vous voudrez probablement aussi lui indiquer que vos résultats n'utilisent pas l'alpha prémultiplié

const canvas = document.querySelector('#c');
const renderer = new THREE.WebGLRenderer({
  canvas,
  alpha: true,
+  premultipliedAlpha: false,
});

Three.js définit par défaut le canvas pour utiliser premultipliedAlpha: true mais définit par défaut les matériaux pour sortir premultipliedAlpha: false.

Si vous souhaitez mieux comprendre quand utiliser l'alpha prémultiplié et quand ne pas l'utiliser, voici un bon article à ce sujet.

Dans tous les cas, mettons en place un exemple simple avec un canvas transparent.

Nous avons appliqué les paramètres ci-dessus à l'exemple de l'article sur la réactivité. Rendons également les matériaux plus transparents.

-  const material = new THREE.MeshPhongMaterial({color});

+    color,
+    opacity: 0.5,
+  });

Faire de votre arrière-plan une animation three.js

Une question fréquente est de savoir comment faire en sorte qu'une animation three.js serve d'arrière-plan à une page web.

Il existe 2 manières évidentes.

  • Définir la propriété CSS position du canvas à fixed comme ceci :
#c {
 position: fixed;
 left: 0;
 top: 0;
 ...
}

Vous pouvez voir exactement cette solution dans l'exemple précédent. Il suffit de définir z-index à -1 et les cubes apparaîtront derrière le texte.

Un petit inconvénient de cette solution est que votre JavaScript doit s'intégrer à la page, et si vous avez une page complexe, vous devez vous assurer qu'aucun des JavaScript de votre visualisation three.js n'entre en conflit avec le JavaScript faisant d'autres choses sur la page.

  • Utiliser un iframe

C'est la solution utilisée sur la page d'accueil de ce site.

Dans votre page web, insérez simplement un iframe, par exemple :

<iframe id="background" src="responsive.html">
<div>
  Votre contenu va ici.
</div>

Puis stylisez l'iframe pour qu'elle remplisse la fenêtre et soit en arrière-plan, ce qui est essentiellement le même code que nous avons utilisé ci-dessus pour le canvas, sauf que nous devons également définir border à none puisque les iframes ont une bordure par défaut.

#background {
    position: fixed;
    width: 100%;
    height: 100%;
    left: 0;
    top: 0;
    z-index: -1;
    border: none;
    pointer-events: none;
}
