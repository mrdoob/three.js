Plusieurs Canevas Plusieurs Scènes

Une question fréquente est de savoir comment utiliser THREE.js avec plusieurs canevas. Supposons que vous souhaitiez créer un site de commerce électronique ou une page avec de nombreux diagrammes 3D. À première vue, cela semble facile. Il suffit de créer un canevas partout où vous voulez un diagramme. Pour chaque canevas, créez un Renderer.

Cependant, vous découvrirez rapidement que vous rencontrez des problèmes.

  1. Le navigateur limite le nombre de contextes WebGL que vous pouvez avoir.

    Typiquement, cette limite est d'environ 8. Dès que vous créez le 9ème contexte, le plus ancien sera perdu.

  2. Les ressources WebGL ne peuvent pas être partagées entre les contextes

    Cela signifie que si vous voulez charger un modèle de 10 Mo dans 2 canevas et que ce modèle utilise 20 Mo de textures, votre modèle de 10 Mo devra être chargé deux fois et vos textures seront également chargées deux fois. Rien ne peut être partagé entre les contextes. Cela signifie également que les choses doivent être initialisées deux fois, les shaders compilés deux fois, etc. Cela empire à mesure qu'il y a plus de canevas.

Alors, quelle est la solution ?

La solution est un seul canevas qui remplit la zone d'affichage en arrière-plan et un autre élément pour représenter chaque canevas "virtuel". Nous créons un seul Renderer puis une Scene pour chaque canevas virtuel. Nous vérifierons ensuite les positions des éléments du canevas virtuel et s'ils sont à l'écran, nous dirons à THREE.js de dessiner leur scène au bon endroit.

Avec cette solution, il n'y a qu'un seul canevas, nous résolvons donc les problèmes 1 et 2 ci-dessus. Nous ne rencontrerons pas la limite de contexte WebGL car nous n'utiliserons qu'un seul contexte. Nous ne rencontrerons pas non plus les problèmes de partage pour les mêmes raisons.

function makeScene(elem) {
  const scene = new THREE.Scene();

  const fov = 45;
  const aspect = 2;  // par défaut le canevas
  const near = 0.1;
  const far = 5;
  const camera = new THREE.PerspectiveCamera(fov, aspect, near, far);
  camera.position.z = 2;
  camera.position.set(0, 1, 2);
  camera.lookAt(0, 0, 0);

  {
    const color = 0xFFFFFF;
    const intensity = 1;
    const light = new THREE.DirectionalLight(color, intensity);
    light.position.set(-1, 2, 4);
    scene.add(light);
  }

  return {scene, camera, elem};
}

function setupScene1() {

Et ensuite, nous allons créer une fonction pour rendre chaque scène uniquement si l'élément est à l'écran. Nous pouvons dire à THREE.js de ne rendre qu'une partie du canevas en activant le test de cisaillement avec Renderer.setScissorTest puis en définissant à la fois le cisaillement et la zone d'affichage avec Renderer.setViewport et Renderer.setScissor.

function renderSceneInfo(sceneInfo) {
  const {scene, camera, elem} = sceneInfo;

  // obtenir la position de cet élément par rapport à la zone d'affichage
  const {left, right, top, bottom, width, height} =
      elem.getBoundingClientRect();

  const isOffscreen =
      bottom < 0 ||
      top > renderer.domElement.clientHeight ||
      right < 0 ||
      left > renderer.domElement.clientWidth;

  if (isOffscreen) {
    return;
  }

  camera.aspect = width / height;
  camera.updateProjectionMatrix();

  const positiveYUpBottom = canvasRect.height - bottom;
  renderer.setScissor(left, positiveYUpBottom, width, height);
  renderer.setViewport(left, positiveYUpBottom, width, height);

  renderer.render(scene, camera);
}

Et ensuite, notre fonction de rendu va d'abord effacer l'écran, puis rendre chaque scène.

function render(time) {
  time *= 0.001;

  resizeRendererToDisplaySize(renderer);

  renderer.setScissorTest(false);
  renderer.clear(true, true);
  renderer.setScissorTest(true);

  sceneInfo1.mesh.rotation.y = time * .1;
  sceneInfo2.mesh.rotation.y = time * .1;

  renderSceneInfo(sceneInfo1);
  renderSceneInfo(sceneInfo2);

  requestAnimationFrame(render);
}

Synchronisation

Le code ci-dessus fonctionne, mais il y a un problème mineur. Si vos scènes sont compliquées ou si, pour une raison quelconque, le rendu prend trop de temps, la position des scènes dessinées dans le canevas sera en décalage par rapport au reste de la page.

Si nous donnons une bordure à chaque zone

  display: inline-block;
  width: 5em;

+  border: 1px solid black;
}

Et nous définissons l'arrière-plan de chaque scène

+scene.background = new THREE.Color('red');

-  position: fixed;
+  position: absolute;

Ensuite, nous allons définir la transformation du canevas pour le déplacer de manière à ce que le haut du canevas soit aligné avec le haut de la partie de la page actuellement défilée.

function render(time) {
  ...

  const transform = `translateY(${window.scrollY}px)`;
  renderer.domElement.style.transform = transform;

position: fixed empêchait le canevas de défiler du tout pendant que le reste de la page défilait par-dessus. position: absolute permettra au canevas de défiler avec le reste de la page, ce qui signifie que tout ce que nous dessinons restera avec la page pendant le défilement, même si nous sommes trop lents pour rendre. Lorsque nous aurons enfin l'occasion de rendre, nous déplacerons le canevas pour qu'il corresponde à l'endroit où la page a été défilée, puis nous effectuerons un nouveau rendu. Cela signifie que seuls les bords de la fenêtre montreront momentanément des parties non rendues, mais le contenu au milieu de la page devrait correspondre et ne pas glisser. Voici une vue des résultats de la nouvelle méthode ralentie 10 fois.

Rendre cela plus générique

Maintenant que nous avons fait fonctionner plusieurs scènes, rendons cela un peu plus générique.

Nous pourrions faire en sorte que la fonction de rendu principale, celle qui gère le canevas, contienne simplement une liste d'éléments et leur fonction de rendu associée. Pour chaque élément, elle vérifierait si l'élément est à l'écran et, si c'est le cas, appellerait la fonction de rendu correspondante. De cette façon, nous aurions un système générique où les scènes individuelles ne seraient pas vraiment conscientes d'être rendues dans un espace plus petit.

Voici la fonction de rendu principale

const sceneElements = [];
function addScene(elem, fn) {
  sceneElements.push({elem, fn});
}

function render(time) {
  time *= 0.001;

  resizeRendererToDisplaySize(renderer);

  renderer.setScissorTest(false);
  renderer.setClearColor(clearColor, 0);
  renderer.clear(true, true);
  renderer.setScissorTest(true);

  const transform = `translateY(${window.scrollY}px)`;
  renderer.domElement.style.transform = transform;

  for (const {elem, fn} of sceneElements) {
    // obtenir la position de cet élément par rapport à la zone d'affichage
    const rect = elem.getBoundingClientRect();
    const {left, right, top, bottom, width, height} = rect;

    const isOffscreen =
        bottom < 0 ||
        top > renderer.domElement.clientHeight ||
        right < 0 ||
        left > renderer.domElement.clientWidth;

    if (!isOffscreen) {
      const positiveYUpBottom = renderer.domElement.clientHeight - bottom;
      renderer.setScissor(left, positiveYUpBottom, width, height);
      renderer.setViewport(left, positiveYUpBottom, width, height);

      fn(time, rect);
    }
  }

  requestAnimationFrame(render);
}

Vous pouvez voir qu'il boucle sur sceneElements, qui est censé être un tableau d'objets, chacun ayant une propriété elem et fn.

Il vérifie si l'élément est à l'écran. Si c'est le cas, il appelle fn et lui passe l'heure actuelle et son rectangle.

Maintenant, le code de configuration de chaque scène s'ajuste simplement à la liste des scènes

    camera.aspect = rect.width / rect.height;
    camera.updateProjectionMatrix();
    mesh.rotation.y = time * .1;
    renderer.render(scene, camera);
  });
}
