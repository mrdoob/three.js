Optimiser l'animation de nombreux objets

Cet article est la suite de un article sur l'optimisation de nombreux objets . Si vous ne l'avez pas encore lu, veuillez le lire avant de continuer.

Dans l'article précédent, nous avons fusionné environ 19 000 cubes en une seule géométrie. Cela présentait l'avantage d'optimiser notre rendu de 19 000 cubes, mais avait l'inconvénient de rendre plus difficile le déplacement de chaque cube individuel.

Selon ce que nous cherchons à accomplir, il existe différentes solutions. Dans ce cas, traçons plusieurs ensembles de données et animons les transitions entre eux.

La première chose à faire est d'obtenir plusieurs ensembles de données. Idéalement, nous pré-traiterions probablement les données hors ligne, mais dans ce cas, chargeons 2 ensembles de données et générons-en 2 autres.

Générons 2 ensembles de données supplémentaires. L'un indiquant les endroits où le nombre d'hommes est supérieur à celui des femmes et inversement, les endroits où le nombre de femmes est supérieur à celui des hommes.

La première chose à faire est d'écrire une fonction qui, étant donné un tableau bidimensionnel de tableaux comme précédemment, va le parcourir pour générer un nouveau tableau bidimensionnel de tableaux.

function mapValues(data, fn) {
  return data.map((row, rowNdx) => {
    return row.map((value, colNdx) => {
      return fn(value, rowNdx, colNdx);
    });
  });
}

Comme la fonction standard Array.map, la fonction mapValues appelle une fonction fn pour chaque valeur du tableau de tableaux. Elle lui transmet la valeur ainsi que les indices de ligne et de colonne.

Maintenant, créons du code pour générer un nouveau fichier qui est une comparaison entre 2 fichiers.

function makeDiffFile(baseFile, otherFile, compareFn) {
  let min;
  let max;
  const baseData = baseFile.data;
  const otherData = otherFile.data;
  const data = mapValues(baseData, (base, rowNdx, colNdx) => {
    const other = otherData[rowNdx][colNdx];
      if (base === undefined || other === undefined) {
        return undefined;
      }
      const value = compareFn(base, other);
      min = Math.min(min === undefined ? value : min, value);
      max = Math.max(max === undefined ? value : max, value);
      return value;
  });
  // make a copy of baseFile and replace min, max, and data
  // with the new data
  return {...baseFile, min, max, data};
}

Le code ci-dessus utilise mapValues pour générer un nouvel ensemble de données qui est une comparaison basée sur la fonction compareFn passée en paramètre. Il suit également les résultats min et max de la comparaison. Enfin, il crée un nouveau fichier avec toutes les mêmes propriétés que baseFile, sauf avec de nouvelles valeurs pour min, max et data.

Notez qu'il y a quelques points de données étranges qui ressortent vraiment. Je me demande ce qu'il en est !??! En tout cas, comment animer entre ces 4 ensembles de données.

Beaucoup d'idées.

  • Effectuer simplement un fondu entre eux en utilisant Material.opacity

    Le problème avec cette solution est que les cubes se chevauchent parfaitement, ce qui entraînera des problèmes de z-fighting. Il est possible que nous puissions résoudre cela en modifiant la fonction de profondeur et en utilisant le blending. Nous devrions probablement l'étudier.

  • Agrandir l'ensemble que nous voulons voir et réduire les autres ensembles.

    Parce que toutes les boîtes ont leur origine au centre de la planète, si nous les réduisons en dessous de 1.0, elles s'enfonceront dans la planète. Au début, cela semble être une bonne idée, mais le problème est que toutes les boîtes de faible hauteur disparaîtront presque immédiatement et ne seront pas remplacées tant que le nouvel ensemble de données ne s'agrandira pas jusqu'à 1.0. Cela rend la transition peu agréable. Nous pourrions peut-être résoudre cela avec un shader personnalisé sophistiqué.

  • Utiliser les Morphtargets

    Les Morphtargets sont un moyen de fournir plusieurs valeurs pour chaque sommet de la géométrie et de "morpher" ou interpoler (interpolation linéaire) entre elles. Les Morphtargets sont le plus souvent utilisés pour l'animation faciale de personnages 3D, mais ce n'est pas leur seule utilisation.

Nous sommes également censés appeler TWEEN.update à chaque frame dans notre boucle de rendu, mais cela met en évidence un problème. "tween.js" est conçu pour un rendu continu, mais nous rendons à la demande. Nous pourrions passer au rendu continu, mais il est parfois agréable de rendre uniquement à la demande, car cela cessera d'utiliser l'énergie de l'utilisateur lorsque rien ne se passe, donc voyons si nous pouvons le faire animer à la demande.

Nous allons créer un TweenManager pour nous aider. Nous l'utiliserons pour créer les Tweens et les suivre. Il aura une méthode update qui retournera true si nous devons l'appeler à nouveau, et false si toutes les animations sont terminées.

class TweenManger {
  constructor() {
    this.numTweensRunning = 0;
  }
  _handleComplete() {
    --this.numTweensRunning;
    console.assert(this.numTweensRunning >= 0);
  }
  createTween(targetObject) {
    const self = this;
    ++this.numTweensRunning;
    let userCompleteFn = () => {};
    // créer un nouveau tween et installer notre propre callback onComplete
    const tween = new TWEEN.Tween(targetObject).onComplete(function(...args) {
      self._handleComplete();
      userCompleteFn.call(this, ...args);
    });
    // remplacer la fonction onComplete du tween par la nôtre
    // afin de pouvoir appeler le callback de l'utilisateur s'il en fournit un.
    tween.onComplete = (fn) => {
      userCompleteFn = fn;
      return tween;
    };
    return tween;
  }
  update() {
    TWEEN.update();
    return this.numTweensRunning > 0;
  }
}

J'espère que parcourir ceci vous a été utile. L'utilisation des morphtargets est une technique courante pour déplacer de nombreux objets. Par exemple, nous pourrions donner à chaque cube une position aléatoire dans une autre cible et "morpher" de là vers leurs premières positions sur le globe. Ce pourrait être une façon cool d'introduire le globe.

Ensuite, vous pourriez être intéressé par l'ajout d'étiquettes à un globe, ce qui est abordé dans Aligner des éléments HTML sur la 3D.

Note : Nous pourrions essayer de simplement afficher le pourcentage d'hommes ou de femmes, ou la différence brute, mais étant donné la façon dont nous affichons les informations (des cubes qui poussent à partir de la surface de la terre), nous préférerions que la plupart des cubes soient bas. Si nous utilisions une de ces autres comparaisons, la plupart des cubes seraient à environ la moitié de leur hauteur maximale, ce qui ne donnerait pas une bonne visualisation. N'hésitez pas à changer la fonction amountGreaterThan de Math.max(a - b, 0) à quelque chose comme (a - b) "différence brute" ou a / (a + b) "pourcentage", et vous verrez ce que je veux dire.
